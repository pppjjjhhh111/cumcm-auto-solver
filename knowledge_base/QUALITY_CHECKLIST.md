# Knowledge Base Quality Checklist

## 精修卡通用标准
- 单张卡必须服务 RAG 检索、模型选择、公式生成、图表规划和论文写作中的至少两个环节。
- 内容必须为原创整理，不复制论文、教材、博客或网页原文。
- 中文为主，必要时保留 TOPSIS、ARIMA、KMeans、GM11 等模型名。
- 标题和关键词要覆盖中文常用说法，例如“综合评价”“熵权法”“线性规划”“灵敏度分析”。
- 每张卡应明确回答：什么时候适合用、什么时候不适合用、需要什么数据、输出什么结果、适合什么图表、论文如何表达、常见扣分点是什么。
- 卡片应避免空泛定义，优先给出竞赛小问中的判断规则、输入输出、误区和可复用表达。

## 单个方法卡检查
- 是否只有一个主题。
- 是否有方法类别和关键词。
- 是否说明适用问题和不适用问题。
- 是否说明输入数据要求和输出结果。
- 是否包含基本步骤和核心公式。
- 是否有 Python 实现建议。
- 是否有优点、局限、常见组合和推荐图表。
- 是否包含论文表达模板和常见扣分点。
- 是否包含 LaTeX 核心公式，且变量含义能被 FormulaAgent 识别。
- 是否说明灵敏度或鲁棒性分析方式。
- 是否说明与其他模型的常见组合，而不是孤立介绍算法。
- 是否区分同类模型边界，例如线性规划与整数规划、熵权法与 TOPSIS。

## 论文模板卡检查
- 是否说明适用题型和题目特征。
- 是否给出推荐论文结构。
- 是否列出常用模型组合和推荐图表。
- 是否给出常见模型假设。
- 是否包含数据预处理、模型建立、结果分析、灵敏度分析和模型评价写法。
- 是否能指导 PaperWriterAgent 按题型生成章节，而不是只给通用模板。
- 是否说明摘要、模型假设、结果分析和模型评价中的常见扣分点。
- 是否避免整篇论文式长段落，保留短段模板和结构提示。

## 题型套路卡检查
- 是否说明常见题目特征。
- 是否说明常见小问形式。
- 是否说明适合模型类别和数据需求。
- 是否说明输出要求和易错点。
- 是否给出推荐解题流程。
- 是否说明第一问、第二问、第三问、最后一问在国赛题中的常见分工。
- 是否能从题面关键词路由到评价、预测、优化、分类聚类、仿真、网络图论或混合题型。
- 是否能支持多方案比较中的可实现性、解释性、稳定性、论文表达性评分。

## Notes 卡检查
- 是否保留固定标题：适用场景、核心原则、推荐做法、不推荐做法、可以生成的图表或表格、论文表达模板、质量检查清单。
- 是否包含具体可执行的检查项，而不是只写原则。
- 是否能服务图表规划、摘要生成、灵敏度分析或最终反思。
- 是否明确常见扣分点，例如空泛摘要、无单位图表、无扰动范围、结论与图表不一致。

## RAG 检索检查
- 查询“综合评价”应命中 evaluation 类卡片。
- 查询“熵权法 TOPSIS”应命中评价方法卡。
- 查询“时间序列预测”应命中 prediction 类卡片。
- 查询“灰色预测”应命中 grey_prediction_gm11.md。
- 查询“线性规划 整数规划”应命中 optimization 类卡片。
- 查询“KMeans 聚类”应命中 classification 类卡片。
- 查询“最短路径”应命中 network 类卡片。
- 查询“灵敏度分析”应命中 notes 或 code_patterns。
- 查询“论文摘要”应命中 abstract_writing 或 paper_templates。
- 查询“图表设计”应命中 figure_design。
- 查询“第一问 第二问 第三问 最后一问”应命中 common_subquestion_patterns。
- 查询“题型判断 混合类”应命中 problem_type_router。
- 查询“data_fit_score reportability_score”应命中 modeling_route_selection。
- 查询“0-1 决策 排班 路径”应命中 integer_programming。
- 查询“连续变量 资源分配 约束条件”应命中 linear_programming。
