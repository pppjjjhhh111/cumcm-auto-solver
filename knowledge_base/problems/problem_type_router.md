# 题型路由规则

## 常见题目特征
题型路由用于在读题后快速判断应检索哪类知识、调用哪类模型模板、生成哪类公式和图表。竞赛题常把评价、预测、优化、分类聚类、仿真、网络图论混合在同一背景下，因此不能只看题目名称，要综合关键词、数据形态、输出动词和约束强度。若题面出现“优劣、水平、综合评价”，优先考虑评价；出现“未来、趋势、缺失补全”，优先考虑预测；出现“最优、调度、配置、路径、成本最低”，优先考虑优化；出现“类型、分组、识别”，优先考虑分类聚类。

## 常见小问形式
评价类常问“建立指标体系并排序”“判断影响程度”；预测类常问“预测未来若干期”“估计需求或风险”；优化类常问“给出最优方案”“在约束下分配资源”；分类聚类类常问“识别类别”“划分等级或群体”；仿真类常问“模拟演化过程”“比较不同规则下结果”；网络图论类常问“路径、连通性、传播、选址或流量”；混合类则先评价或预测，再把结果作为优化输入。

## 适合模型类别
评价类关键词包括“综合、排名、水平、权重、打分”，适合熵权法、层次分析、TOPSIS、灰色关联、主成分分析，不适合直接用黑箱预测替代排序，输出为权重、得分、排名和雷达图。预测类关键词包括“趋势、未来、估计、预测、外推”，适合 ARIMA、指数平滑、灰色 GM、回归、随机森林、LSTM 的简化实现，不适合样本极少时使用高参数深度模型，输出为预测值、置信区间、误差指标和趋势图。优化类关键词包括“最大、最小、约束、调度、配置”，适合线性规划、整数规划、多目标规划、动态规划、启发式搜索，不适合没有明确目标函数时强行优化，输出为决策变量、目标值、约束满足情况和方案表。

分类聚类类关键词包括“类型、等级、识别、分群、相似”，适合 K-means、层次聚类、DBSCAN、逻辑回归、决策树、SVM，不适合在无标签数据上声称监督分类准确率，输出为类别标签、特征贡献、混淆矩阵或聚类画像。仿真类关键词包括“演化、传播、排队、随机、蒙特卡洛”，适合元胞自动机、排队模型、系统动力学、蒙特卡洛和离散事件仿真，不适合把单次模拟当稳定结论，输出为多轮统计、情景对比和收敛图。网络图论类关键词包括“节点、边、路径、连通、最短路、最大流”，适合最短路径、最小生成树、最大流、中心性、社区发现和选址模型，不适合忽略边权、方向或容量，输出为路径、关键节点、网络指标和拓扑图。混合类适合分阶段建模，输出为路由说明、阶段结果和最终综合方案。

## 数据需求
路由前要识别数据是截面、时间序列、空间坐标、文本标签、网络边表还是仿真参数。评价类需要指标正负向和权重来源；预测类需要时间粒度、历史长度和验证区间；优化类需要目标函数、决策变量、上下界和约束；分类聚类需要特征矩阵、标签或聚类数依据；仿真需要初始状态、转移规则和随机分布；网络图论需要节点、边、权重、方向、容量和坐标。

## 输出要求
路由结果应给出“题型-关键词-典型任务-适合模型-不适合模型-预期输出”的结构化记录，便于 RAG 命中后生成模型候选。最终答案不能只说采用某算法，还要说明为什么该题型匹配、数据是否支持、公式如何写、图表如何展示。多题型时应列出主任务和辅助任务，例如“预测需求后做库存优化”“评价风险后做区域分级”。

## 容易出错的地方
最常见错误是把所有排序题都当评价题，却忽略其中可能包含预测或优化；把“影响因素分析”误判为因果模型；把“分类”和“聚类”混用；优化题没有检查可行域；网络题没有构造正确边权；仿真题没有重复实验和随机种子说明。若题面数据不足，应选择可解释的降级模型，并在论文中说明假设。

## 推荐解题流程
先抽取关键词和输出动词，再判断数据结构；随后生成二到三个候选题型，并按数据适配度、实现难度、解释性和论文表达性打分。确定主路由后，补充一个备用模型用于稳健性比较。建模时先写目标、变量、参数，再写公式和算法流程；求解后用表格承接数值结果，用折线图、柱状图、热力图、网络图或情景对比图展示主要发现。

## 与论文结构的关系
题型路由决定论文方法章节的骨架。评价题重在指标体系和权重解释；预测题重在数据划分、误差评价和趋势解释；优化题重在目标函数、约束和可行方案；分类聚类题重在特征选择和类别画像；仿真题重在规则、参数和多次实验；网络图论题重在图构建和拓扑指标；混合类论文应按阶段组织，避免模型堆砌而缺少主线。
