# 分类代码套路

## 适用场景
有标签分类、模型训练、预测和混淆矩阵。

## 输入
题目数据文件、字段说明、模型参数、输出目录和图表目录。输入路径必须位于项目工作区内。

## 输出
分类报告、混淆矩阵、特征重要性。所有结果应保存到 `outputs/code` 或受控工作目录，图表保存到 `outputs/figures`。

## 主要步骤
1. 读取数据并检查文件是否存在。
2. 统一列名、类型和缺失值处理。
3. 执行模型或统计计算。
4. 保存中间表和最终结果。
5. 生成图表并记录文件路径。
6. 输出简短 stdout，便于执行日志阅读。

## 推荐 Python 库
sklearn, pandas。

## 常见错误
路径写死、列名不存在、空数据表、类型转换失败、图表目录不存在、随机过程未固定种子。

## 容错建议
对缺失文件返回清晰错误；对缺失列给出可用列名；对空数据跳过模型并写 warning；所有异常写入执行日志。

## 结果保存建议
核心 JSON 保存到 `outputs/code/analysis_results.json`，表格保存到 `outputs/code/summary_table.csv`，不要写入用户主目录。

## 图表保存建议
使用稳定文件名，保存到 `outputs/figures`，并在结果 JSON 中记录图表路径。

## 论文对应关系
代码输出的指标进入“模型求解”和“结果分析”，图表进入“数据预处理与探索性分析”或“结果分析”。
