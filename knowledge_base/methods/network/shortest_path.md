# 最短路径

## 方法类别
网络图论类模型，适用于路径规划、流量分配、节点重要性、传播网络、供应链网络。

## 关键词
最短路径、Dijkstra、Floyd、路径规划、交通网络

## 适用问题
节点和边权明确，需要寻找最短距离、时间或成本路径的场景。题面通常包含多个对象、变量或约束，需要把数据处理、模型计算和结果解释连接起来。

## 不适用问题
边权随时间变化或存在容量约束时需扩展模型。若数据质量不足，应先做缺失值、异常值和口径一致性检查，再决定是否使用该方法。

## 输入数据要求
- 需要明确样本、对象或节点的定义。
- 需要字段含义、量纲和方向说明。
- 需要区分数值型、类别型、时间型或网络型数据。
- 若涉及监督学习，需要目标变量或标签。
- 若涉及优化，需要目标函数、决策变量和约束条件。

## 输出结果
- 模型参数、得分、类别、路径、预测值或最优方案。
- 可形成结果表、排序表、误差表或方案对比表。
- 可生成路径示意图、距离矩阵表。
- 论文中应解释数值含义、适用范围和可能偏差。

## 基本步骤
1. 明确问题目标和变量定义。
2. 清洗数据，统一量纲、方向和缺失值处理规则。
3. 构造模型输入矩阵或网络结构。
4. 按方法公式计算核心指标或求解目标。
5. 输出结果表和关键图表。
6. 做误差、灵敏度或鲁棒性检查。
7. 将结论转化为面向题目的解释。

推荐使用 networkx, pandas, matplotlib。实现时应固定随机种子，保存中间表，避免访问项目外路径，图表保存到 `outputs/figures`，结果保存到 `outputs/code`。

- 结果可以通过图表和表格解释。

## 局限
- 对数据预处理和字段含义敏感。
- 若参数、权重或阈值缺少依据，结论说服力会下降。
- 复杂题目中通常需要与其他模型组合使用。

## 常见组合
最小费用流、中心性分析。

## 灵敏度或鲁棒性分析
可对关键权重、参数、阈值、训练集划分或约束上限做扰动，比较结果排名、误差、目标值或类别是否稳定。

## 推荐图表
路径示意图、距离矩阵表。
