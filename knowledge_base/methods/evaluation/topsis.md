# TOPSIS 理想解排序法（熵权 TOPSIS 排序卡）

## 方法类别
TOPSIS 是多指标综合评价与方案排序方法，通过对象到正、负理想解的距离判断优劣。它常与熵权法组成“熵权 TOPSIS”：熵权法客观赋权，TOPSIS 综合排序。

## 关键词
TOPSIS、理想解、负理想解、接近度、距离测度、综合排序、正向化、标准化、熵权 TOPSIS、排名稳定性。

## 适用问题
适用于方案、地区、年份、企业或策略的优劣比较，尤其是题目要求“综合评价、排名、选优、分级、推荐方案”时。它可解释性强，便于论文写作。

## 不适用问题
若指标方向不明、权重无依据、对象不可比，TOPSIS 排名会失真。若核心是预测、因果识别、路径规划或动态控制，TOPSIS 只能辅助评价。

## 输入数据要求
输入为评价矩阵 $X=(x_{ij})_{m\times n}$ 和权重向量 $w$。需说明指标含义、单位、来源和方向。成本型转为越大越优，区间型按偏离目标程度正向化。权重可来自熵权法、AHP 或组合赋权。

## 输出结果
输出加权标准化矩阵、正理想解 $A^+$、负理想解 $A^-$、距离 $D_i^+$ 与 $D_i^-$、接近度 $C_i$、排名和等级。论文中应解释靠前对象的优势与短板。

## 基本步骤
1. 构造指标矩阵并处理缺失值、异常值。
2. 对成本型和区间型指标正向化。
3. 标准化数据，消除量纲影响。
4. 确定权重；熵权 TOPSIS 中先用熵权法求 $w_j$。
5. 构造加权标准化矩阵，求正负理想解。
6. 计算距离、接近度和排名。
7. 扰动权重或标准化方式，检验排序稳定性。

## 核心公式
标准化：
$$z_{ij}=\frac{y_{ij}}{\sqrt{\sum_{i=1}^{m}y_{ij}^2}}$$

加权矩阵：
$$v_{ij}=w_jz_{ij},\quad \sum_{j=1}^{n}w_j=1$$

正负理想解：
$$A^+=\{\max_i v_{ij}\},\quad A^-=\{\min_i v_{ij}\}$$

距离与接近度：
$$D_i^+=\sqrt{\sum_{j=1}^{n}(v_{ij}-A_j^+)^2},\quad D_i^-=\sqrt{\sum_{j=1}^{n}(v_{ij}-A_j^-)^2}$$
$$C_i=\frac{D_i^-}{D_i^+ + D_i^-},\quad 0\le C_i\le 1$$
接近度越大，说明对象越接近理想状态，排名越靠前。

## Python 实现建议
用 pandas 组织指标表、权重表和结果表；用 numpy 计算标准化、理想解和距离。建议拆分 `normalize`、`entropy_weight`、`topsis` 三个函数，输出 `D_plus`、`D_minus`、`C`、`rank`。

## 优点
流程直观，能同时考虑接近最优和远离最差；结果可解释，适合排名表、柱状图、雷达图。与熵权法组合后，权重和排序都有公式支撑。

## 局限
结果依赖指标选择、标准化方式和权重；极端值会改变理想解；默认指标之间可补偿，可能掩盖某些硬性短板。

## 常见组合
熵权法 + TOPSIS 最常用；AHP + TOPSIS 可体现专家偏好；熵权-AHP + TOPSIS 可平衡主客观因素；聚类 + TOPSIS 可先分组再排序。

## 灵敏度或鲁棒性分析
重点扰动权重、异常值处理和标准化方法。可将高权重指标上下调整 5% 到 20%，重算 $C_i$ 和排名；也可比较等权、熵权、AHP 权重下的 Top-K。若关键排名变化小，则结论稳定。

## 论文表达模板
“本文在完成指标正向化和标准化后，采用熵权法确定客观权重，并将其代入 TOPSIS 模型。通过构造加权标准化矩阵，计算各评价对象与正、负理想解的距离及相对接近度，最终依据接近度完成综合排序。”

## 常见扣分点
未先正向化就求理想解；标准化口径与权重口径不一致；权重来源不清；只列排名不解释接近度含义；没有进行权重扰动或排名稳定性分析。
